XerahS MCP Server

Use XerahS as a first-class MCP tool for agents and editor integrations instead of shelling out to ad hoc scripts.

Discovery

The public manifest is published at https://xerahs.com/.well-known/mcp/manifest.json. Remote deployments use https://mcp.xerahs.com/mcp/ and https://mcp.xerahs.com/mcp/events/.

Authentication

HTTP transport uses a bearer token stored by XerahS. The key is generated automatically the first time the MCP runtime starts if one does not already exist.

Supported Tools

  • capture_region, capture_window, capture_full_screen, and capture_scrolling
  • annotate_image for headless annotation rendering
  • upload_file and upload_clipboard
  • query_history and get_history_item
  • list_workflows and get_settings

Supported Resources

  • History items, latest capture, and history search results
  • Workflows and workflow details
  • Capture, upload, history, general, and integration settings
  • Configured destinations and monitor inventory

Local Usage

xerahs-mcp --mcp
xerahs-mcp --mcp-server --transport http --port 7890

Behavior Notes

  • The MCP server boots the real XerahS runtime headlessly.
  • annotate_image writes a new file and does not launch the interactive editor.
  • History IDs are row IDs from the SQLite history store, represented as strings.
  • Only MCP-safe settings are exposed; uploader secrets are not returned.
